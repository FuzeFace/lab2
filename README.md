# Лабораторная работа 2 — Классификация видео

## Цель работы

Получить практический опыт работы с видеоданными и реализовать систему классификации видео с использованием современных методов извлечения признаков.

В данной работе решается задача классификации видео из подмножества **HowTo100M | Hobbies**.  
В качестве признаков используются векторы **VideoPrism**, а для классификации обучается отдельная модель.

---

## Постановка задачи

Задача видеоклассификации заключается в автоматическом определении категории видеоролика.  
Современные видео содержат как визуальную информацию об объектах, так и динамику их изменения во времени.

В рамках лабораторной работы необходимо было:

- выполнить предобработку видеоданных;
- извлечь кадры из видео;
- нормализовать и изменить размер изображений;
- получить векторные признаки видео;
- обучить модель классификации;
- оценить качество модели;
- построить графики и confusion matrix;
- выполнить анализ ошибок.

## Предобработка видео

Для каждого видеоролика выполнялись следующие этапы:

1. Загрузка видео по `video_id`.
2. Извлечение фиксированного количества кадров.
3. Приведение кадров к размеру `288 × 288`.
4. Нормализация RGB-значений в диапазон `[0, 1]`.
5. Передача кадров в VideoPrism.

Использованные параметры:

| Параметр | Значение |
|---|---:|
| Количество кадров | 16 |
| Размер кадра | 288 × 288 |
| Нормализация | `[0, 1]` |

---

## Извлечение признаков

В качестве визуального энкодера использовалась модель:

```text
videoprism_public_v1_base
```

VideoPrism применялся как замороженный извлекатель признаков.  
Это означает, что веса VideoPrism не обучались, а использовались только для получения векторного представления каждого видео.

Схема получения признаков:

```text
Видео → Кадры → VideoPrism → Токены признаков → Mean Pooling → Вектор видео
```

Полученные VideoPrism-векторы использовались как вход для обучаемого классификатора.

---

## Архитектура модели

Поверх VideoPrism-векторов была обучена MLP-модель.

Архитектура классификатора:

```text
Linear(D → 512)
LayerNorm
GELU
Dropout(0.35)

Linear(512 → 256)
LayerNorm
GELU
Dropout(0.35)

Linear(256 → 5)
```

Количество выходных нейронов соответствует количеству классов:

```text
5 классов
```

## Анализ результатов

На тестовой выборке модель достигла accuracy:

```text
0.55
```

И F1 Macro:

```text
0.545
```

Для небольшой выборки из 200 видео и 5 визуально близких классов результат можно считать приемлемым.

Наиболее уверенно модель распознавала классы:

- Crafts
- Toys
- Games

Наиболее сложным оказался класс:

```text
Tricks and Pranks
```

Для него получен самый низкий F1-score:

```text
0.43
```

Это можно объяснить тем, что видео категории `Tricks and Pranks` могут визуально пересекаться с другими категориями, например с `Toys`, `Games` или `Crafts`.

## Анализ ошибок

Для анализа ошибок были сохранены дополнительные файлы:

| Файл | Описание |
|---|---|
| [`test_predictions.csv`](./results1/test_predictions.csv) | Все предсказания модели на test-выборке |
| [`misclassified_examples.csv`](./results1/misclassified_examples.csv) | Только ошибочные предсказания |

Ошибки классификации связаны с тем, что некоторые категории внутри Hobbies имеют похожие визуальные признаки.

Например, видео из категорий `Games`, `Toys` и `Tricks and Pranks` могут содержать похожие объекты, людей, игровые действия или бытовые сцены.

Также важно учитывать, что для каждого видео использовался короткий фрагмент. Из-за этого часть полезной информации могла не попасть в выбранные кадры.

## Возможные улучшения

Качество классификации можно улучшить следующими способами:

1. Увеличить количество видео в каждом классе.
2. Использовать более длинные видеофрагменты.
3. Извлекать больше кадров из каждого видео.
4. Использовать несколько фрагментов из одного видео.
5. Добавить текстовые признаки из субтитров.
6. Использовать более сложную классификационную модель.
7. Применить Top-3 Accuracy вместо Top-5 Accuracy.
8. Дообучать часть video encoder, а не только классификационную голову.
